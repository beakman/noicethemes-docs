La plantilla incluye inglés (`en`) como idioma predeterminado y español (`es`) como idioma secundario. Las páginas en inglés no tienen prefijo de URL (`/about`); las páginas en español llevan el prefijo `/es/` (`/es/contacto`).

---

## Archivos de traducción

Todas las cadenas visibles por el usuario viven en archivos JSON bajo `src/locales/`:

```
src/locales/
├── en/
│   ├── common.json    ← etiquetas de navegación, pie de página, banner de cookies, alternador de tema
│   ├── main.json      ← hero, estadísticas, about, testimonios
│   └── sections.json  ← características, precios, FAQ, contacto, galería, etc.
└── es/
    ├── common.json
    ├── main.json
    └── sections.json
```

Estos archivos se cargan automáticamente mediante `import.meta.glob` en `src/i18n/ui.ts`.

### Formato de las claves

Las claves de traducción usan el formato `namespace:ruta.anidada`:

```ts
t("common:nav.ariaLabel")     // → "Navegación principal"
t("main:hero.headline")       // → "Diseño que<br />habla por sí solo"
t("sections:features.heading") // → "Todo lo que necesitas"
```

El namespace coincide con el nombre del archivo (`common`, `main`, `sections`). La ruta es la ruta de claves JSON anidadas.

## Usar traducciones en componentes

```astro
---
import { useTranslations } from "@i18n/utils";

const { lang } = Astro.props; // "en" | "es"
const t = useTranslations(lang);

const heading = t("main:hero.headline") as string;
const features = t("sections:features.items") as Array<{ title: string; description: string }>;
---

<h1 set:html={heading} />
{features.map(f => <div>{f.title}</div>)}
```

Los arrays y objetos pueden convertirse a sus tipos esperados — TypeScript inferirá `unknown` por defecto.

## Rutas traducidas

El enrutamiento i18n de Astro requiere que los nombres de las rutas coincidan en cada idioma. Una página de "contacto" en español vive en `/es/contacto`, no en `/es/contact`. El mapeo se define en `src/i18n/routes.ts`:

```ts
export const routes: Record<string, Record<string, string>> = {
  es: {
    about:   "sobre-nosotros",
    contact: "contacto",
    blog:    "blog",
    pricing: "precios",
  },
};
```

Usa `useTranslatedPath(lang)` para generar URLs correctas independientemente del idioma:

```ts
import { useTranslatedPath } from "@i18n/utils";
const translatePath = useTranslatedPath(lang);

translatePath("/contact") // → "/contact" (en) o "/es/contacto" (es)
translatePath("/blog")    // → "/blog"    (en) o "/es/blog"    (es)
```

## Selector de idioma

El componente `LanguageSwitcher` en el pie del menú móvil lee la URL actual, mapea el slug al slug del idioma destino mediante `routes.ts` y renderiza enlaces a ambas versiones del idioma. Se genera por página en `Navbar.astro`:

```ts
const langSwitchUrls = (Object.keys(languages) as Array<"en" | "es">).map((code) => {
  // mapea el slug de la ruta actual al slug del idioma destino
  ...
  return { code, label: languages[code], url };
});
```

## Añadir una cadena de traducción

1. Añade la clave a `src/locales/en/common.json` (o `main.json` / `sections.json`).
2. Añade la traducción al español en el archivo correspondiente en `src/locales/es/`.
3. Llama a `t("namespace:tu.nueva.clave")` en tu componente.

```json
// src/locales/en/common.json
{
  "cta": {
    "bookCall": "Book a free call"
  }
}
```

```json
// src/locales/es/common.json
{
  "cta": {
    "bookCall": "Reservar una llamada gratuita"
  }
}
```

## Añadir un tercer idioma

1. Añade el idioma a `src/i18n/ui.ts`:

```ts
export const languages = {
  en: "English",
  es: "Español",
  fr: "Français",   // ← añade
};
```

2. Crea `src/locales/fr/common.json`, `main.json` y `sections.json` con las traducciones en francés.

3. Añade los mapeos de rutas a `src/i18n/routes.ts`:

```ts
export const routes = {
  es: { contact: "contacto", /* ... */ },
  fr: { contact: "contact",  about: "a-propos", /* ... */ },
};
```

4. Replica todas las páginas bajo `src/pages/fr/` siguiendo el mismo patrón que `src/pages/es/`.

5. Actualiza `astro.config.mjs` para incluir el nuevo idioma:

```ts
i18n: {
  defaultLocale: "en",
  locales: ["en", "es", "fr"],
}
```
